2-IR / Line_Follower_2IR.ino Embedded C++

How it works: Two digital IR sensors read the line position. If both detect the line (LOW), the robot moves forward. If only the left sensor sees the line, the robot turns right to correct. If only the right sees it, turn left. If neither sees the line, stop.

// Define IR sensor pins
#define LEFT_IR_PIN  7
#define RIGHT_IR_PIN 8

#define MOTOR1_PIN1 3  // L293D input 1
#define MOTOR1_PIN2 2  // L293D input 2
#define MOTOR2_PIN1 5
#define MOTOR2_PIN2 4

void loop() {
  int leftSensor  = digitalRead(LEFT_IR_PIN);
  int rightSensor = digitalRead(RIGHT_IR_PIN);

  if (leftSensor == LOW && rightSensor == LOW) {
    moveForward();   // Both on line → go straight
  }
  else if (leftSensor == LOW && rightSensor == HIGH) {
    turnRight();     // Left on line → correct right
  }
  else if (leftSensor == HIGH && rightSensor == LOW) {
    turnLeft();      // Right on line → correct left
  }
  else {
    stopMoving();    // Both off line → stop
  }
}
3-IR-L293D-Shield / Maze_Solver_3IR.ino Embedded C++

How it works: Three analog IR sensors read the line with an Adafruit Motor Shield. During exploration, the robot records turns as 'L', 'R', 'S' (straight), 'U' (U-turn). After exploration, it optimizes the path by simplifying LUR sequences (e.g., LUS → R, LUL → S). The optimized path is then replayed at speed.

#include <AFMotor.h>

char a[100];  // exploration path
char b[100];  // optimized path
int mode = 0;

void loop() {
  if (mode == 0) {  // EXPLORE mode
    int l = digitalRead(LS);
    int r = digitalRead(RS);
    int m = digitalRead(MS);

    if (l == 1 && m == 0 && r == 1) {     // Straight
      a[i++] = 'S';
    } else if (l == 0 && m == 0 && r == 1) { // Left turn
      a[i++] = 'L';
    } else if (l == 1 && m == 0 && r == 0) { // Right turn
      a[i++] = 'R';
    }
  }
  else {  // SOLVE mode — path optimization
    // Simplify LUR sequences:
    // LUS → R,  LUL → S,  LUR → U
    if (a[j] == 'L' && a[j+1] == 'U' && a[j+2] == 'S')
      b[q++] = 'R';  j += 3;
    else if (a[j] == 'L' && a[j+1] == 'U' && a[j+2] == 'L')
      b[q++] = 'S';  j += 3;
    // ... more optimization rules ...
  }
}
Line-Followers-And-Obstacle-Avoidance / Line_Follower_Obstacle_Sweep.ino Embedded C++

How it works: A 2-state machine (FOLLOW_LINE / AVOIDING) with a 14-step sub-machine for avoidance. When an obstacle is detected at ≤5cm, the robot executes a sequence of turns and forward movements. The key innovation is the sweep step (#12) that alternates left/right at 70ms intervals, scanning for the line until it's reliably detected for 120ms.

enum State { FOLLOW_LINE, AVOIDING };
State currentState = FOLLOW_LINE;
int avoidStep = 0;

void loop() {
  if (currentState == FOLLOW_LINE) {
    if (getDistance() < OBSTACLE_DISTANCE) {
      currentState = AVOIDING;
      avoidStep = 0;
      return;
    }
    followLine();
  }
  else {
    avoidSequence();
  }
}

void avoidSequence() {
  switch (avoidStep) {
    case 0: turnRight(); break;  // Turn away from obstacle
    case 2: moveForward(); break; // Move alongside
    case 4: turnLeft(); break;   // Turn back toward line
    case 6: moveForward(); break; // Pass obstacle
    case 8: turnLeft(); break;   // Align with line
    case 12: // SWEEP: actively scan for line
      sweepForLine();
      break;
    case 13: // Final right turn, return to FOLLOW_LINE
      turnRight();
      currentState = FOLLOW_LINE;
      break;
  }
}
RoboQuest / Obstacle_Avoiding.ino Embedded C++

How it works: A 7-state machine with non-blocking millis() timers. When an obstacle is detected at ≤12cm, the robot stops, turns right away from the line, moves forward alongside the obstacle, continues until the ultrasonic sensor reads >18cm (clear), then turns left back toward the line. The REACQUIRE state validates line detection for 250ms before resuming.

enum State {
  FOLLOW_LINE, AVOID_STOP, AVOID_TURN_OUT,
  AVOID_GO_PARALLEL, AVOID_PASS_OBSTACLE,
  AVOID_TURN_IN, REACQUIRE
};

void loop() {
  switch (state) {
    case FOLLOW_LINE:
      if (distanceCM() <= 12) {
        enterState(AVOID_STOP);
      }
      followLine();
      break;

    case AVOID_PASS_OBSTACLE:
      moveForward();
      if (distanceCM() > 18) {
        enterState(AVOID_TURN_IN);
      }
      break;

    case REACQUIRE:
      moveForward();
      // Require stable line for 250ms before locking
      if (seen && millis() - lineSeenStart >= 250) {
        enterState(FOLLOW_LINE);
      }
      // Timeout at 1800ms → gentle spin search
      if (elapsed(1800)) {
        turnLeft(); delay(250);
        enterState(REACQUIRE);
      }
      break;
  }
}
RoboQuest / State_Round / Objective_1.ino Embedded C++

How it works: A 10-state machine that handles a complete competition course: line follow → timed stops (2s/3s) → obstacle detection at ≤17cm → more line following → bottle zone approach → final stop. Each state transitions to the next on completion, creating a fully autonomous competition run.

// 10-state objective machine for RoboQuest
enum State {
  FOLLOW_LINE,    STOP_2s,     FOLLOW_LINE_2,
  STOP_3s,        FOLLOW_LINE_3,
  OBSTACLE_CHECK, STOP_3s_2,   FOLLOW_LINE_4,
  FINAL_STOP_1s,  BOTTLES_PRE_CIRCLE,
  BOTTLES_IN_CIRCLE
};

void loop() {
  switch (state) {
    case FOLLOW_LINE:
      followLine();
      if (bothSensorsOff()) {
        enterState(STOP_2s);
      }
      break;

    case STOP_2s:
      stopMoving();
      delay(2000);  // Timed pause
      enterState(FOLLOW_LINE_2);
      break;

    case OBSTACLE_CHECK:
      // Check ultrasonic distance
      if (distanceCM() <= 17) {
        // Obstacle detected — handle it
      }
      enterState(STOP_3s_2);
      break;
  }
}
Intersection-Handler / Intersection-Handler.ino Embedded C++

How it works: A 4-state machine that detects intersections (both sensors off-line). When this happens, the robot stops for 2 seconds, moves forward 0.5 seconds past the intersection, then resumes line following. At the next intersection, it stops permanently. This is useful for grid-based tracks with junctions.

enum State {
  FOLLOW_LINE, INTERSECTION,
  FOLLOW_LINE_AGAIN, STOP
};

void loop() {
  switch (state) {
    case FOLLOW_LINE:
      if (bothOffLine()) {
        state = INTERSECTION;
        stopMoving();
        delay(2000);
        moveForward();
        delay(500);
        state = FOLLOW_LINE_AGAIN;
      } else {
        followLine();
      }
      break;

    case FOLLOW_LINE_AGAIN:
      if (bothOffLine()) {
        state = STOP;  // Final stop at 2nd intersection
      } else {
        followLine();
      }
      break;

    case STOP:
      stopMoving();  // Permanently stopped
      break;
  }
}

Hardware & Tech Stack

The components that brought the code to life.

Arduino Uno / Nano

ATmega328P microcontroller — 16 MHz, 32 KB flash, running Embedded C++ via Arduino IDE.

IR Sensors

2–4 digital/analog infrared reflectance sensors. LOW = on line (black absorbs IR), HIGH = off line.

HC-SR04 Ultrasonic

40 kHz sonar rangefinder. 2 cm – 4 m range, used for obstacle detection and parking spot finding.

L293D Motor Driver

Dual H-bridge for bi-directional DC motor control. Drives two motors with up to 600 mA per channel.

Adafruit Motor Shield

PWM-based motor driver shield for Arduino. Used in the 3-IR maze solver for smoother motor control.

LDR + LED

Light-dependent resistor for ambient light sensing. Activates headlight LED in dark tunnels (analogRead < 450).
